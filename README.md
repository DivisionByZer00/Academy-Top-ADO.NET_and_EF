# Academy-Top-ADO.NET_and_EF

#### Решения домашних заданий "Технология доступа к базам данных ADO.NET и EF"

## Homework 01

### Тема: "Анатомия GUI приложения. Введение в Avalonia UI"

**Задание:** Вам необходимо спроектировать и реализовать структуру базы данных для небольшой библиотеки. Программа не требует написания C# кода; ваша задача — подготовить SQL-скрипт, который создает таблицы с правильными связями и наполняет их первичными данными.

**Техническое задание:** 

Таблица Genres (Жанры): Спроектируйте таблицу-справочник. Она должна хранить уникальный идентификатор (Id) и название жанра (Name). Название не должно быть пустым и должно быть уникальным.

Таблица Books (Книги): Спроектируйте таблицу для хранения книг. Каждая книга должна иметь:    
- Уникальный артикул/ID;
- Название (обязательно);
- Автор;
- Цену (число с плавающей точкой);
- Связь с жанром (GenreId).

Ограничения (Constraints):
- Установите внешний ключ (Foreign Key) между книгами и жанрами.
- Настройте правило: при удалении жанра, у связанных книг поле жанра должно становиться пустым (SET NULL).

**Совет по выполнению:**    
Вы можете использовать любой текстовый редактор (например, VS Code) для последовательного написания всех команд в один файл. После того как скрипт будет готов, вы можете файл целиком в SQL Editor в SQLiteStudio (Ctrl + O – запустить скрипты из файла) и запустить одной командой. Это гарантирует, что ваша база будет создана именно в той последовательности, которую вы задумали.

## Homework 02

### Тема: "Чтение данных: SELECT, JOIN и Агрегаты"

**Задание:** Используя базу данных «Библиотека», созданную в первом домашнем задании, вам необходимо подготовить SQL-скрипт для формирования аналитической отчетности. Скрипт должен содержать серию запросов, которые позволяют извлечь и обработать данные для нужд администратора библиотеки.

**Техническое задание:**
Ваш итоговый файл должен содержать решения для следующих аналитических задач:    
- ***Формирование витрины данных***: Выведите названия всех книг и их авторов. Используйте псевдонимы (AS), чтобы в результате заголовки столбцов назывались Название_Книги и Автор_Книги. Добавьте расчетный столбец Цена_с_Наценкой, где цена увеличена на 15%.
- ***Поиск по маске:*** Найдите все книги, в названии которых есть слово «Программирование» или «C#».
- ***Аудит данных:*** Выведите список всех книг, у которых не указан жанр (поле GenreId равно NULL).
- ***Связывание данных (JOIN):*** Напишите запрос, объединяющий таблицы Books и Genres. В результате должны быть столбцы: Название книги, Автор и Название жанра.
- Выполните сначала INNER JOIN (чтобы увидеть только книги с жанрами).
- Затем выполните LEFT JOIN (чтобы увидеть абсолютно все книги, даже без жанров).
- ***Агрегация и группировка:*** Посчитайте количество книг в каждом конкретном жанре. Итоговая таблица должна содержать два столбца: Название жанра и Количество.
- ***Фильтрация групп:*** Выведите названия жанров, у которых средняя цена книг превышает 1000 единиц. Отсортируйте результат по убыванию цены.
- ***Пагинация:*** Напишите запрос, который имитирует получение данных для «второй страницы» каталога (вывести 3 книги, пропустив первые 3), отсортировав их по названию.

**Совет по выполнению:**    
Вы можете использовать любой текстовый редактор (например, VS Code) для последовательного написания всех команд в один файл. После того как скрипт будет готов, вы можете файл целиком в SQL Editor в SQLiteStudio (Ctrl + O – запустить скрипты из файла) и запустить одной командой. Это гарантирует, что ваша база будет создана именно в той последовательности, которую вы задумали.
